# LDO

LDO=low dropout regulator，低压差 + 线性 + 稳压器。

“低压差”：输出压降比较低，例如输入 3.3V，输出可以达到 3.2V。

“线性”：LDO 内部的 MOS 管工作于线性电阻。

“稳压器” 说明了 LDO 的用途是用来给电源稳压。

LDO 芯片，内部为一个 P-MOS + 一个运放 + 2 个电阻

因此 LDO 核心架构：P-MOS + 运放，通过芯片内部已经设置好的电阻来达到调节 P-MOS 的输出，而得到该芯片的输出电压。

LDO 工作原理就一句话：通过运放调节 P-MOS 的输出。

# 地址对齐

地址不对齐，程序会跑飞。

在初始化的使用字节对齐时，只要存在非 32 位的初始化，必然存在某个变量的数据存放时跨字节的。例如：初始化某个变量为 int16，那么它的占用空间为两个字节，存储时极其有可能第一个字节在上一个四个字节中，第二个字节在下一个字节中。32 位 arm 芯片是 4 个字节取址，当出现这种情况的时候，就会出现断裂，程序一次只能取四个字节，超过就跑飞了。

# Keil 文件类型概述

# 1、Project Files 工程类型文件

工程类型文件是比较重要的，特别是工作空间、 工程文件、工程选项配置这三种文件不能删除。项目界面布局文件可以删除。

# 1.1 工作空间文件

（这类文件不能删除） .uvmpw：µVision Project file for Multiple Projects 多工程文件，也就是我们说的工作空间文件。

# 1.2 工程文件

（这类文件不能删除） .uvprojx：µVision5 工程文件

.uvproj：µVision4 工程文件

.uv2：µVision3 工程文件

# 1.3 工程选项配置文件

（这类文件不能删除） .uvoptx：µVision5 工程选项配置文件

.uvopt：µVision4 工程选项配置文件

# 1.4 项目界面布局文件

（这类文件可以删除） .uvguix [.user-name]：µVision5 项目界面布局文件。删除之后，重新打开工程，界面布局会恢复到默认布局。如 Demo.uvguix.Administrator。

.uvguix [.user-name]：µVision4 项目界面布局文件。

# 2、Source Files 源代码类型文件

这一类文件相信大家都熟悉，简述一下。

.c：C 源文件

.h：C 头文件

.cpp：C++ 源文件

.s：汇编源文件

.a51：汇编源文件

.a66：汇编源文件

.inc：汇编头文件

.src：C 编译器生成的其他源文件

# 3、Listing Files 链接类型文件

这一类文件属于中间文件，一般在编译过程中产生。可以在工程目标选项中配置对应的链接文件。Project -> Options for Target -> Listing，如下图：

.map：存储镜像文件 这个文件比较重要，我会在后面专门讲述一篇 map 相关文章。

.i：C 预处理器输出文件

.scr：链接器分散加载器文件

.lst：C 编译器或汇编器生成的文件

.m51：同样也是 map 文件

.m66：同样也是 map 文件

.htm：链接器的列表文件

# 4、Object and HEX Files 目标和 Hex 类型文件

这一类文件同样属于中间文件，一般在编译过程中产生。但这类文件比较重要，调试信息、预览信息、可执行文件等都在这里面。

Project -> Options for Target -> Output，如图：

.hex：可执行文件

.axf：包含调试信息的程序文件 有时候我们调试出错，可能是 axf 没有文件。

.d：编译生成的依赖文件 一般一个.c 文件对应一个.d 文件。

.o：也是目标的依赖文件 重定位定址的依赖文件。

.lib：库文件 生成的库就在这里。

.elf：ELF/DWARF 链接的文件

.crf：浏览信息文件 使用 Go to Definition Of 功能，就需要有这类文件。

# 5、Build Debugger 编译调试文件

编译和调试，其实是两类不同文件，由于比较少，综合在一起写。

.bat：批处理文件

._IA,\ .__I, ._II,\ .SCR：工具调用文件

.ini：于初始化源码文件

# 7、Other Files 其他类型文件

这类文件比较多，但使用较少，简单列几个吧。

.pack：软件（支持）包文件

.sct：链接控制文件

.lnp：连接器传递命令文件

.dep：目标编译依赖文件

.cdb：µVision 设备数据库文件

# 瑞萨芯片一些操作细节

需要把 keil 的工程的配置文件删除，避免无法通过仿真下载程序（瑞萨芯片目前不能通过 ram 仿真器（jtag）在 keil 下直接下载）该文件的后缀是：.uvoptx

# 低功耗要注意的一些问题

理论上没有用的 io 要配置成模拟输入 若是配置成输出 应要拉高与 mcu 内部电源电压持平 像这种情况耗电很小是微安级别 如果 io 与外部电路存在压差 那么就是 ma 级别

有用的 io 休眠时 不能有压差
